# Lean Evaluation Plan

## Goal

Measure whether eligibility decisions are consistent, explainable, and useful for engineering iteration.

## Label tiers

1. `rule_based_label` (silver): generated by deterministic checks at scale.
2. `llm_assisted_label` (silver+): used for text ambiguity resolution.
3. `human_review_label` (gold): manually reviewed hard cases only (target 15-25).

## Metrics

- eligibility classification: precision, recall, F1, accuracy,
- citation quality: percent of decisions with at least one evidence item,
- rule coverage: percent of criteria evaluated deterministically.

## Error taxonomy

- `retrieval_miss`: protocol evidence not found,
- `criteria_parse_error`: trial text could not be normalized,
- `reasoning_conflict`: contradictory justification vs output,
- `missing_data`: patient fields insufficient to evaluate criterion.

## Reporting standard

Each run should emit:
- aggregate metrics,
- confusion matrix counts (`tp`, `tn`, `fp`, `fn`),
- top error buckets with example case IDs.
